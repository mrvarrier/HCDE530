# week 2 reflection: code literacy and documentation

## what code literacy means
being able to read and explain what a block of code does. making technical stuff understandable for both technical and non-technical people.

## my background
i have a cs undergrad so python concepts like csv handling, data structures, functions, and loops were already familiar. this week was more of a refresher since it's been a while since i've written python code.

## what i demonstrated

### reading code
i read and analyzed the demo_word_count.py script to understand how it loads csv files, processes text data, and calculates statistics. identified what the code does and where it could be clearer for beginners.

### working with claude
claude wrote all the detailed educational comments and documentation, not me. my role was directing what needed to be documented, specifying the target audience (hcde students who are beginners), and reviewing the outputs for accuracy.

### technical understanding
demonstrated understanding of csv file operations, text processing with string methods like .split(), basic statistics (min, max, average), and data visualization with html/css/javascript. also understood how different encodings, missing data, and large files could break the script in real-world scenarios.

### documentation approach
initially had claude create verbose educational comments explaining both what and why for teaching purposes. later simplified to match my personal coding style - concise, lowercase, minimal comments that are brief but clear.

## modern code literacy for hcd
code literacy isn't just about writing code yourself. it's about understanding what code does, directing ai tools to generate solutions, evaluating technical quality, and translating between technical and non-technical stakeholders. this reflects how hcd practitioners actually work with code - analyzing, directing, and reviewing rather than always coding from scratch.

## key insight
demonstrated code literacy through analysis, direction, and review. understood the technical concepts from my cs background, specified what documentation was needed for teaching, and collaborated with ai tools to create educational materials. the skill is knowing how to work with code and technical systems even when you're not the one writing every line.
